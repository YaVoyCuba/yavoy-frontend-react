## 🛠️ Tech Stack

- **Node.js** 18+
- **TypeScript** 5+ (strict mode enabled)
- **React** 18+ (functional components + hooks)
- **Build Tool** Vite
- **Routing** React Router v6
- **Styling** Tailwind CSS

- **Package Manager** pnpm (npm as fallback)
- **Testing** Vitest + React Testing Library + Playwright
- **Backend** REST API (Laravel)

## 📋 Code Quality Standards

### TypeScript
- ✅ Strict mode: ALL tsconfig.json requirements
- ✅ No implicit `any` (explain if necessary)
- ✅ Discriminated unions for state machines
- ✅ Zod validation for API responses
- ✅ Generic constraints for reusable types

### React
- ✅ Functional components only
- ✅ Components < 200 lines
- ✅ React.memo() for expensive renders
- ✅ useCallback for stable callbacks
- ✅ useMemo for derived state
- ✅ Custom hooks for reusable logic
- ✅ Code splitting with React.lazy + Suspense
- ✅ Error boundaries for graceful degradation

### Testing
- ✅ 80%+ coverage for components with state
- ✅ 90%+ coverage for custom hooks
- ✅ 70%+ coverage for API integration
- ✅ E2E tests for critical user flows
- ✅ Use Vitest + React Testing Library + Playwright

### Performance
- ✅ JavaScript bundle < 200KB gzipped per route
- ✅ Lighthouse score 90+
- ✅ First Contentful Paint < 1.8s
- ✅ Largest Contentful Paint < 2.5s
- ✅ Cumulative Layout Shift < 0.1

### Styling
- ✅ Tailwind CSS (primary)
- ✅ Mobile-first responsive design
- ✅ Semantic HTML
- ✅ WCAG AA accessibility standard
- ✅ Focus states visible
- ✅ Sufficient color contrast

## 🔌 Backend Integration

### API Configuration
- Endpoint: `VITE_APP_API_URL` (from `.env.local`)
- Authentication: Bearer token in Authorization header
- Content-Type: `application/json`

### Response Validation
```typescript
// All API responses validated with Zod
const UserSchema = z.object({
  id: z.string().uuid(),
  name: z.string(),
  email: z.string().email(),
});

export async function fetchUser(id: string) {
  const data = await apiCall(`/users/${id}`);
  return UserSchema.parse(data); // Validates and returns typed data
}
```

### Error Handling
- Custom APIError type with statusCode and responseText
- 401 errors → redirect to login
- Retry logic: exponential backoff (1s, 2s, 4s) for 408/429/5xx
- Structured logging: timestamp, error, statusCode, context

## 🚀 Development Commands

```bash
# Setup
pnpm install

# Development
pnpm dev              # Start Vite dev server

# Linting & Formatting
pnpm lint             # Run ESLint
pnpm format           # Format with Prettier

# Type Checking
pnpm type-check       # TypeScript type checking (tsc --noEmit)

# Testing
pnpm test             # Run Vitest
pnpm test --watch     # Watch mode
pnpm test:coverage    # Coverage report
pnpm exec playwright test  # E2E tests

# Building
pnpm build            # Production build
pnpm build --profile  # Build with analysis
pnpm preview          # Preview prod build locally

# Analysis
pnpm analyze          # Bundle size analysis
pnpm lighthouse       # Lighthouse audit
```

## 🆘 Troubleshooting

| Issue | Solution |
|---|---|
| TypeScript errors | Run `pnpm type-check` to see all errors |
| Lint failures | Run `pnpm format` to auto-fix most issues |
| Tests failing | Run `pnpm test --watch` and debug |
| Build errors | Check `pnpm build` output, clear cache: `rm -rf node_modules/.vite` |
| Node modules corrupted | `rm -rf node_modules pnpm-lock.yaml && pnpm install` |
